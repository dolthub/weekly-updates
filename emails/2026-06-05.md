### BranchBench

A team at Columbia University thinks [agents need branches](https://arxiv.org/abs/2604.17180). So much so that they proposed a new benchmark called BranchBench and measure Dolt and Neon on it. Obviously, we agree and were excited to [review their findings](https://www.dolthub.com/blog/2026-06-03-branch-bench-database-benchmarking-for-agentic-workflows/)! 

Dolt did great on the version control part of the tasks but struggled on the query path for a couple queries. We identified two optimizations in our SQL engine and will have those patched ASAP. If you are trying Dolt and find a slow query, we will fix it! Dolt is built from the ground up so we have a ways to go before our SQL engine is optimization free. 

[For simple queries we're faster than MySQL](https://www.dolthub.com/docs/sql-reference/benchmarks/latency/) so we know we can get there.

### Dumbo Garbage Collection

We again implore you to let us know if you're interested in version-controlled MongoDB. We're going to keep talking about it regardless. This week [Dumbo launched garbage collection](https://www.dolthub.com/blog/2026-06-02-dumbodb-gc/). As some of you old timers who've been using Dolt since before it had automated garbage collection know, Dolt's storage engine makes a lot of disk garbage. Garbage collection is essential for any moderate scale use case. Dumbo now has it. Bring us your use case.

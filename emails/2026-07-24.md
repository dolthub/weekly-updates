### Beads Server Mode

As some of you know, Dolt is the database behind [Beads](https://github.com/gastownhall/beads), one of the most popular [agentic memory](https://www.dolthub.com/blog/2026-01-22-agentic-memory/) products. Dolt's Beads integration has undergone some back and forth. First embedded, then server, and back to embedded. This week, [Dustin](https://www.dolthub.com/team#dustin) announced [a hardened server implementation](https://www.dolthub.com/blog/2026-07-22-introducing-beads-proxied-server-mode), which we think will eventually become the default. 

Beads is getting fairly mature, it's easy to use, and it helps your agents work for longer. If you've been waiting to try it, now might be the time.

### Try Ito

I met the CTO of [Ito](https://www.ito.ai/) at an event and he convinced me to try it on Doltgres. Ito is a "runtime analysis code review tool". Ito asked us to call it that. I prefer "AI QA Agent". 

But anyways, you hook Ito up to your GitHub Pull Requests and you get a report of bugs it finds for your PR. Ito doesn't review code. Ito runs your product and uses the diffs to inform its testing workflow. It's a pretty cool, novel application of AI. [Jason's](https://www.dolthub.com/team#jason) review [details some real bugs it helped us find](https://www.dolthub.com/blog/2026-07-20-ito-ai-qa-for-doltgresql).

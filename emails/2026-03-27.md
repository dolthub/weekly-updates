### Announcing DoltLite

I spent [a week in Gas Town](https://www.dolthub.com/blog/2026-03-24-a-week-in-gas-town) and all I got was this amazing [DoltLite](https://www.dolthub.com/blog/2026-03-25-doltlite). 

[DoltLite](https://github.com/timsehn/doltlite) is a fork of SQLite where the [B-tree](https://en.wikipedia.org/wiki/B-tree) storage backend is replaced with a Dolt-inspired [Prolly Tree](https://docs.dolthub.com/architecture/storage-engine/prolly-tree). DoltLite is [entirely vibe-coded using Gas Town](https://www.dolthub.com/blog/2026-03-24-a-week-in-gas-town/). DoltLite is as if SQLite and Git had a baby — a drop-in replacement for SQLite with Git-style version control features.

DoltLite is currently under my personal GitHub organization. DoltLite is first and foremost an experiment with supporting a purely vibe coded application. But, with enough interest, we'll move it under the DoltHub organization and throw the full weight of our company behind it. Reply to this email if you have a use case. We already have two users I know of.

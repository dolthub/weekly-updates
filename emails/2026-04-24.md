### Dueling Claude and Cursor

We're all trying to figure out the best way to use coding agents. Vibe Coding [DoltLite](https://github.com/dolthub/doltlite) has taught me a lot. I have some pro tips. 

[Gas Town was a great way to get started](https://www.dolthub.com/blog/2026-03-24-a-week-in-gas-town/), allowing me to reach the prototype phase in less than a day. But, we're building a database here. All that parallelism became a burden as I got into hand-to-hand combat with agents fixing gnarly issues. I settled on [a dueling agents strategy](https://www.dolthub.com/blog/2026-04-22-dueling-agents/): claude in one terminal and codex in another. Claude is good at new features. Codex is good at review. You can use them both effectively together.

### Doltgres MCP

We're driving hard to [Doltgres 1.0](https://www.dolthub.com/blog/2025-10-16-state-of-doltgres/#looking-forward-to-10) which will land later this year. Doltgres is already quite good and the install base is growing quite rapidly. A user recently asked for our [Dolt MCP server](https://github.com/dolthub/dolt-mcp) to work with Doltgres. Within a week we obliged. [Dolt MCP server supports Doltgres](https://www.dolthub.com/blog/2026-04-23-doltgres-mcp-server/). 

In fairness, we were already on it because we want [agent mode in Workbench](https://www.dolthub.com/blog/2026-02-09-introducing-agent-mode/) to work with Doltgres. Agent mode in Workbench is my favorite feature of the year and you all know how I feel about DoltLite.

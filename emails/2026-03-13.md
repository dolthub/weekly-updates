### Multi-Agent Persistence

Many of you are hearing about Dolt for the first time via [Beads](https://github.com/steveyegge/beads), [Gas Town](https://github.com/steveyegge/gastown), or [Wasteland](https://steve-yegge.medium.com/welcome-to-the-wasteland-a-thousand-gas-towns-a5eb9bc8dc1f). How did Dolt end up as the persistence layer for Beads and thus powering Gas Town and Wasteland? 

My [latest article](https://www.dolthub.com/blog/2026-03-13-multi-agent-persistence/) explains how Dolt is perfect for multi-agent persistence. It turns out multi-agent systems require version-controlled structured data accessed concurrently. Dolt is the only database that does that.

### Hosted Dolt Branch Permissions

[Hosted Dolt](https://hosted.doltdb.com/) comes with a built-in SQL workbench. A customer wanted to restrict direct changes on `main` and only allow changes to `main` via Pull Request, much like a common workflow on GitHub. We had this feature on DoltHub for a few years but Hosted Dolt is online so the feature was more involved to implement. But, we managed to do it and launch the [feature this week](https://www.dolthub.com/blog/2026-03-12-hosted-branch-permissions/).

### Agent Mode Gets Better

Since launching agent mode in Dolt Workbench, we've been adding improvements based on what we've learned from using it ourselves and hearing from you. Eric wrote about [the latest enhancements](https://www.dolthub.com/blog/2026-03-19-agent-mode-features/) we've shipped over the past month. The updates make agent mode more practical for actual database work - better handling of complex queries, improved context awareness, and more reliable results. If you tried agent mode early on and found some rough edges, worth checking out again.

### SSH Remotes

[You can now push and pull Dolt databases using SSH](https://www.dolthub.com/blog/2026-03-17-ssh-remotes/). We've supported HTTPS and file-based remotes for a while, but teams kept asking for SSH support so they could use the same key infrastructure they already have set up. Neil wrote the announcement explaining how it works. If you're managing Dolt databases across multiple servers and already have SSH keys configured everywhere, this should make your life easier.

### lazydolt

If you've used [lazygit](https://github.com/jesseduffield/lazygit), you know how nice it is to have an interactive terminal UI for Git operations instead of typing commands. Now there's [lazydolt](https://www.dolthub.com/blog/2026-03-16-lazydolt/), which brings the same approach to Dolt databases. Zach wrote about it. It's a Terminal User Interface built in Go that lets you navigate branches, view diffs, manage commits, and run database operations without leaving your terminal. The interesting bit is that it was built mostly by AI coding agents, which is fitting given how much we've been talking about AI and databases lately.

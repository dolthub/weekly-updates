### Function Indexes

A long standing feature request in both Dolt and Doltgres was function indexes. The common use case for function indexes is to add an index on the lower case of a column to do fast lookups when you are unsure of casing of that column. Another more advanced use case is indexes into parts of a JSON document. Well, [function indexes are here for Dolt](https://www.dolthub.com/blog/2026-04-29-announcing-functional-indexes-in-dolt/) and I just saw [a PR for Doltgres](https://github.com/dolthub/doltgresql/pull/2642). Function indexes in Doltgres should be ready some time next week.

### Incremental Garbage Collection

We continue to improve Dolt garbage collection. Many of you probably haven't noticed because automatic garbage collection has been on by default for about a year, but Dolt makes a lot of disk garbage. On large imports, you could get into a situation where your host did not have enough memory to collect the garbage. [Incremental garbage collection](https://www.dolthub.com/blog/2026-04-28-introducing-incremental-garbage-collection/) gives you an out in this case. Incremental garbage collection is off by default. We want to turn it on by default so we're testing it on internal Dolt databases just to be sure it is safe.

### DoltHub API V2

Did you know that DoltHub has an API? You can use the API to run SQL, create pull requests, or resolve issues programmatically. The `v1alpha1` version grew organically from feature requests and was tightly coupled to our GraphQL implementation. This week, [we shipped `v2`](https://www.dolthub.com/blog/2026-07-09-dolthub-api-v2), which was actually designed with proper error codes, better asynchronous interface, and thorough testing. 

[The Wasteland](https://steve-yegge.medium.com/welcome-to-the-wasteland-a-thousand-gas-towns-a5eb9bc8dc1f) makes heavy use of the DoltHub API so if you're looking for decentralized database use cases that might be an inspiration.

### Dolt vs DoltLite Storage

Dolt and DoltLite have different storage engines. They both use the same algorithms but the code and implementation specifics are different. [My latest article](https://www.dolthub.com/blog/2026-07-08-dolt-doltlite-storage-comp) explains the exact differences.

### Dumbo Getting Better

You're going to keep hearing about [Dumbo](https://github.com/dolthub/dumbodb) until [Neil](https://www.dolthub.com/team#neil) gets tired of writing about it. Dumbo is our MongoDB-compatible version of Dolt. 

For those of you unfamiliar with MongoDB, it's a very popular [document database](https://en.wikipedia.org/wiki/Document-oriented_database) where every stored object is JSON. This structure means you can be more fast-and-loose with schema compared to relational databases like MySQL or Postgres. With the rise of Javascript, MongoDB became a popular backend in web stacks. If I need JSON on the frontend, why not store data in that format too? We think version control helps all databases including document databases.

In the past couple weeks, Neil [made indexes work](https://www.dolthub.com/blog/2026-06-30-dumbodb-indexes) and also [explained how to paginate results](https://www.dolthub.com/blog/2026-07-07-dumbodb-log-pagination). We have a long way to go and we're still looking for early Dumbo users.
